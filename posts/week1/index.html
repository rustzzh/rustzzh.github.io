芯片与指令集

之所以会想要了解一下这个方面主要是因为这周拿着m1pro对着“精品开源”项目sonic编译不过去产生的。

注：从一月十六开始就有人在说需要支持苹果芯片，六月了还没写:(

产生了这样一个问题：为什么同一份代码在A上可以编译，在B上就无法编译了呢？

虽然学了计算机体系结构，但指令集、芯片、微架构的含义基本一个都说不出来。

首先解释概念：

  • 指令集(Instruction Set Architecture)：指令集是一组指令的集合，这组指令可以看作对硬件的操作集合。上层应用编写者（包括但不限于汇编语言开发者、写编译器的、应用开发者）所能对机器进行的所有操作就是这个指令集合所描述的。因此可以将指令集看作硬件暴露出来的操作接口。
  • 芯片(CPU)：CPU是执行指令的硬件。
  • 微架构(Microarchitecture)：微架构是指令集在特定CPU上的实现方式，可以将其看作是CPU和ISA的中间层，负责将ISA中的指令转换成CPU能够执行的机器码。

在有了基础概念之后尝试对问题作出回答：代码中使用了某些B平台指令集所不支持的指令。为了避免产生不可预知的问题，go在代码中支持条件编译，可以用于限定某些代码只能在特定版本、特定操作系统上进行编译。

// +build linux darwin 能够在linux、mac上进行编译

// +build go1.15 !go1.16 能够在go1.15版本编译，无法在go1.16版本编译
